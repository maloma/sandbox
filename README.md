# FamilyPilot Public Validation

PF-08 now tests one module at a time after four real participants reported that the broad prototype was not understandable.

## Active Participant Test

### M1 — Operations and Categories

`https://maloma.github.io/sandbox/`

Checks only:

- expense, income and transfer selection;
- amount, category, optional description and date;
- adding a new category inside the operation flow;
- immediate selection of the new category;
- basic operation history;
- short contextual help and optional deeper explanation;
- mobile readability without horizontal scrolling or pinch zoom;
- privacy-safe result sharing.

Participant tasks:

1. add an expense;
2. create and use a custom category;
3. add an income;
4. finish and share the result.

## Planned Modules — Not Active for Participants

- M2 — debts and historical opening balances;
- M3 — obligations;
- M4 — savings and protected reserve;
- M5 — safe-to-spend;
- M6 — capital, FX and crypto;
- M7 — integrated journey after module passes.

`debt.html` is an internal early draft and is not an active participant test. It must not be distributed until it supports a clear recovery path when a repayment is greater than the tracked debt balance.

## Evidence Boundary

The first broad prototype received four real participant responses. All four reported that the overall purpose and required actions were unclear. The broad prototype is therefore retired from participant testing.

Those responses remain valid disconfirming evidence for the integrated journey, but they are not mixed with the new M1 evidence series.

Current validation decision:

`CHANGE — TEST ONE MODULE AT A TIME`
